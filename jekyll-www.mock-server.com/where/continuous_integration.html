Both Travis CI and drone.io are used for continuous integration.

Currently Travis CI and drone.io are both being used to determine which platform works better for MockServer. The conclusion so far is that drone.io builds are more reliable however with Travis CI it is easier to deploy artifacts to a secured repository.

Travis CI   

The build uses Java 1.7 to build a SNAPSHOT version which is deployed to Sonatype.

Although normal MockServer release are build using Java 1.6 it is not possible to use Java 1.7 on Travis CI due to the minimum version of Maven that is supported.

For information on how to use the bleading edge SNAPSHOT version see Sonatype SNAPSHOT

drone.io   

The build uses a docker container to create a clean and separated build environment for each build. The drone.io builds use Java 1.8 but do not deploy there output to any repository.
